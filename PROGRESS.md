# 📊 项目进度追踪与协作者同步机制

> "单一事实来源（Single Source of Truth）"

---

## 🎯 问题识别

### 当前问题

1. **进度描述不一致**
   - MEMORY.md 说 85%
   - 项目计划.md 说 70%
   - idea-box.md 可能又有其他数字
   - **问题：** 没有统一的进度定义和追踪

2. **信息孤岛**
   - AliceDesigner（文档）不知道 Claude（代码）的进度
   - Claude（代码）不知道 AliceBussiness（美工）的进度
   - **问题：** 协作者之间缺少同步机制

3. **完成标准模糊**
   - "完成"是指设计完成？实现完成？还是测试完成？
   - **问题：** 没有 Definition of Done (DoD)

4. **文档更新滞后**
   - 代码/美工已完成，但文档还显示"待完成"
   - **问题：** 缺少自动/手动的进度更新机制

---

## 📋 解决方案

### 1. 单一事实来源（Single Source of Truth）

创建 `PROGRESS.md` 作为**唯一**的进度追踪文档，其他文档的进度数据应引用此文件。

**文件位置：** `demon_village_game/PROGRESS.md`

**更新频率：** 每次 commit 时更新

**负责人：** 各协作者完成工作后立即更新

### 3. Definition of Done (DoD) - 完成标准

#### 设计文档完成标准
- [ ] 文档已创建并写入 `docs/design/`
- [ ] 内容经过审核（项目所有者确认）
- [ ] 已提交到 Git 仓库
- [ ] 进度更新到 `PROGRESS.md`

#### 代码实现完成标准
- [ ] 代码已实现并写入 `code/`
- [ ] 通过基础测试（Godot 运行无报错）
- [ ] 已提交到 Git 仓库
- [ ] 进度更新到 `PROGRESS.md`

#### 美术资源完成标准
- [ ] 资源已创建并写入 `code/assets/`
- [ ] 符合规格要求（尺寸、格式、风格）
- [ ] 已提交到 Git 仓库
- [ ] 进度更新到 `PROGRESS.md`

### 4. 协作者同步机制

#### 每日站会（异步）

**时间：** 每日 23:00（GMT+8）

**形式：** 在 `daily_logs/` 创建当日日志

**模板：**
```markdown
# 2026-04-02 工作日志

## 协作者：AliceDesigner

### 今日完成
- [x] 完成 10 个 NPC 对话树设计
- [x] 提交到 GitHub

### 明日计划
- [ ] 补充战斗系统数值设计
- [ ] 审核代码实现进度

### 需要帮助
- 无

### 进度更新
- 对话系统设计：100% → PROGRESS.md 已更新
```

#### 每周同步（实时）

**时间：** 每周日 20:00（GMT+8）

**形式：** 项目所有者召集，各协作者汇报

**议程：**
1. 各协作者汇报本周进度（5 分钟/人）
2. 讨论阻塞问题（10 分钟）
3. 确定下周目标（5 分钟）
4. 更新 `PROGRESS.md`（5 分钟）

---

### 5. Git 提交规范

**格式：**
```
<type>(<scope>): <subject>

<body>

<footer>
```

**示例：**
```
feat(dialogue): 完成陈默对话树

- 28 个对话节点
- 4 个信任阶段
- 3 个特殊节点

进度更新：PROGRESS.md 已更新
```

**类型说明：**
- `feat` - 新功能
- `fix` - 修复
- `docs` - 文档
- `progress` - 进度更新

---

### 6. 进度更新自动化（可选）

#### GitHub Actions 自动更新

创建 `.github/workflows/progress.yml`：

```yaml
name: Update Progress

on:
  push:
    branches: [main]

jobs:
  update-progress:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      
      - name: Update PROGRESS.md
        run: |
          # 自动统计各模块文件数量
          # 更新进度百分比
          # 提交更改
```

#### 简单方案：手动更新模板

创建 `scripts/update_progress.sh`：

```bash
#!/bin/bash
# 统计各模块文件数量
DESIGN_FILES=$(ls docs/design/*.md | wc -l)
CODE_FILES=$(ls code/scripts/*.gd | wc -l)
ASSET_FILES=$(ls code/assets/*.png | wc -l)

# 计算进度
DESIGN_PROGRESS=$((DESIGN_FILES * 10))
CODE_PROGRESS=$((CODE_FILES * 5))

# 更新 PROGRESS.md
echo "设计进度：$DESIGN_PROGRESS%"
echo "代码进度：$CODE_PROGRESS%"
```

## 🔄 协作者工作流

### AliceDesigner（文档设计）

1. 创建设计文档 → `docs/design/`

# 每日工作进展记录

本目录用于记录项目的每日开发进展、问题追踪和重要决策。

## 目录结构

为了区分不同协作者的工作记录，本目录包含以下子目录：

```
daily_logs/
├── claude/              # Claude (AI Agent) 的工作记录
│   └── YYYY-MM-DD.md
├── remote_agent/        # 远端 Agent 的工作记录
│   └── YYYY-MM-DD.md
└── README.md            # 本说明文件
```

## 记录格式

建议使用以下命名格式：`YYYY-MM-DD.md`（例如：`2026-03-16.md`）

每位协作者在自己的子目录下创建工作记录文件。

## 记录模板

```markdown
# 日期：YYYY-MM-DD

## 今日完成
- [列出今天完成的工作]

## 进行中
- [正在进行的工作]

## 遇到的问题
- [问题描述及解决方案]

## 明日计划
- [明天的开发计划]

## 备注
- [其他需要记录的内容]
```

## 记录规范

1. **及时记录**：每天结束工作时更新进展记录
2. **简洁明了**：用简洁的语言描述工作内容
3. **问题追踪**：记录遇到的问题和解决方法，便于后续参考
4. **计划明确**：为下一步工作制定清晰计划
5. **协作者标注**：在记录中标注工作完成者（Claude / Remote Agent / JCVBoss）

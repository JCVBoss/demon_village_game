## 🔍 发现的问题

### 问题 1：缺少"新协作者快速入门"文档

**现状：**
- 新协作者需要阅读 README → CONTRIBUTING → CONTRIBUTORS → 协作指南 → 澄清说明
- **至少 5 个文档**才能完全理解协作方式

**建议：**
创建一个 `docs/project/新协作者指南.md`，包含：
```markdown
# 新协作者指南

## 第一步：了解项目（10 分钟）
阅读：README.md

## 第二步：找到你的角色（5 分钟）
阅读：CONTRIBUTORS.md（只看"当前贡献者"表格）

## 第三步：领取任务（5 分钟）
1. 访问 Project Board
2. 选择 Todo 任务
3. 在 Issue 中评论"我来做"

## 第四步：开始工作
- 程序员：看 CONTRIBUTING.md 的 Git 流程
- 美术：看 docs/design/美术规范.md（待创建）
- 文档：看 docs/project/文档管理规范.md

## 第五步：写日志
- 程序员：daily_logs/claude/
- 文档：daily_logs/alice/
- 美术：daily_logs/alicebussiness/
```

---

### 问题 2：文档过多，缺乏优先级

**现状：**
`docs/project/` 下有 14 个文档，新协作者不知道该看哪个。

**建议：**
在 `docs/project/README.md` 中添加阅读优先级：
```markdown
# 项目管理文档索引

## 🚨 新协作者必读（P0）
1. 协作方式澄清说明.md — 统一规范，避免混淆
2. 协作指南.md — 详细协作流程

## 📋 按需阅读（P1）
3. 项目管理手册.md — Scrum+Kanban 流程详情
4. 文档管理规范.md — 如果要写文档

## 📚 参考资料（P2）
5. backlog.md — Product Backlog 详情
6. kanban.md — Kanban 快照
7. GitHub_Projects 指南.md — GitHub Projects 使用
...（其他文档标记为"历史报告"或"参考"）
```

### 问题 3：日志目录命名不够直观

**现状：**
```
daily_logs/
├── alice/          # AliceDesigner 用
├── claude/         # Claude 用
├── alicebussiness/ # AliceBussiness 用
```

**新协作者的疑问：**
- "我是新来的，应该创建 `daily_logs/我的名字/` 还是 `daily_logs/我的角色/`？"
- "为什么 AliceDesigner 用 `alice/` 而不是 `alicedesigner/`？"

**建议：**
在 CONTRIBUTING.md 中明确说明：
```markdown
## 每日日志目录

**现有协作者：**
- `alice/` — AliceDesigner（文档设计）
- `claude/` — Claude（代码开发）
- `alicebussiness/` — AliceBussiness（美术设计）

**新协作者：**
请使用你的名字（小写，无空格），例如：
- `daily_logs/zhangsan/`
- `daily_logs/lisi/`

如果不确定，请在 Issue 中询问或创建后在 PR 中说明。
```

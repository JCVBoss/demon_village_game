### 3️⃣ Git 提交 Co-authored-by 格式

**✅ 正确格式：**

```bash
# Claude 提交代码
git commit -m "feat: 实现对话系统

Co-authored-by: Claude <noreply@anthropic.com>"

# AliceDesigner 提交文档
git commit -m "docs: 更新角色设定

Co-authored-by: AliceDesigner <alice-designer@example.com>"

# AliceBussiness 提交美术资源
git commit -m "art: 添加陈默立绘

Co-authored-by: AliceBussiness <alice-bussiness@example.com>"
```

**❌ 错误格式：**
- `Co-authored-by: Remote Agent <...>` — 旧称，已废弃
- `Co-authored-by: Alice <...>` — 不完整，应使用 AliceDesigner

## 🔄 协作流程（简化版）

### 新协作者快速上手

**步骤 1：查看任务**
```
访问：https://github.com/users/JCVBoss/projects/1
查看：Board 视图的 "Todo" 列
```

**步骤 2：领取任务**
```
1. 在 Issue 中评论 "我来做"
2. 将任务从 "Todo" 拖到 "In Progress"
```

**步骤 3：开始工作**
```bash
git pull origin main
git checkout -b feature/任务 ID-描述
# 开发/设计/编写
git add <你的文件>
git commit -m "类型：描述

Co-authored-by: 你的名字 <email>"
git push origin feature/任务 ID-描述
```

**步骤 4：创建 PR**
```
1. GitHub 上创建 Pull Request
2. 关联 Issue：Closes #123
3. 等待审查合并
```

**步骤 5：写日志**
```markdown
# daily_logs/你的名字/YYYY-MM-DD.md

## 今日完成
- [x] 任务 1

## 明日计划
- [ ] 任务 2
```

## ⚠️ 常见误区

### 误区 1：目录名用全称
- ❌ `daily_logs/alicedesigner/`
- ✅ `daily_logs/alice/`

### 误区 2：Co-authored-by 用旧称
- ❌ `Co-authored-by: Remote Agent <...>`
- ✅ `Co-authored-by: AliceDesigner <alice-designer@example.com>`

### 误区 3：不看 Project Board
- ❌ 直接开始做"觉得重要"的事
- ✅ 先从 Project Board 领取任务

### 误区 4：不写日志
- ❌ 默默工作，不记录进展
- ✅ 每天在 `daily_logs/` 写进展

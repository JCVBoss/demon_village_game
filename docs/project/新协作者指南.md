# 🚀 新协作者指南

> 第一次参与「魔王城下的最后村庄」？从这里开始！

**预计用时：** 15 分钟  
**适用人群：** 程序员、美术、文档撰写者、游戏设计师

---

## 🎯 5 步快速开始

### 第 1 步：了解项目（5 分钟）

**阅读：** [README.md](../README.md)

**你会了解到：**
- 这是什么游戏（叙事驱动 RPG）
- 核心特色（LLM 智能体、多结局）
- 项目结构（code/、docs/、daily_logs/）

**不用细读的内容：**
- 目录树形索引（需要时再查）
- 技术细节（后续会接触到）

---

### 第 2 步：找到你的角色（3 分钟）

**阅读：** [CONTRIBUTORS.md](../CONTRIBUTORS.md) 的"当前贡献者"表格

**现有协作者：**
| 角色 | 成员 | 工作目录 |
|------|------|---------|
| 项目所有者 | JCVBoss | 全局 |
| 代码开发 | Claude | `code/` |
| 文档设计 | AliceDesigner | `docs/` |
| 美术设计 | AliceBussiness | `code/assets/` |

**你是哪种协作者？**
- 💻 **程序员** → 负责 `code/` 目录
- 🎨 **美术** → 负责 `code/assets/` 目录
- 📝 **文档/设计** → 负责 `docs/` 目录
- 🎮 **游戏测试** → 测试游戏，报告 Bug

---

### 第 3 步：领取任务（5 分钟）

**访问：** https://github.com/users/JCVBoss/projects/1

**操作：**
1. 查看 "Todo" 列（待办任务）
2. 选择一个适合你的任务
3. 在关联的 Issue 中评论："我来做这个"
4. 将任务从 "Todo" 拖到 "In Progress"

**任务优先级说明：**
- **P0** — 必须有，优先做
- **P1** — 应该有，其次做
- **P2** — 可以有，空闲时做

---

### 第 4 步：开始工作

#### 💻 程序员

**阅读：** [CONTRIBUTING.md](../CONTRIBUTING.md) 的 "Git 工作流" 部分

**基本流程：**
```bash
# 1. 克隆仓库
git clone https://github.com/JCVBoss/demon_village_game.git
cd demon_village_game

# 2. 创建分支
git checkout -b feature/任务 ID-简短描述

# 3. 开发功能（使用 Godot 4.6）

# 4. 提交代码
git add .
git commit -m "feat: 实现功能

Co-authored-by: 你的名字 <your@email.com>"

# 5. 推送
git push origin feature/任务 ID-简短描述
```

**提交信息规范：**
- `feat:` — 新功能
- `fix:` — Bug 修复
- `docs:` — 文档更新
- `refactor:` — 重构

---

#### 🎨 美术设计师

**工作流程：**
1. 从 GitHub 拉取最新代码
2. 将资源文件放入 `code/assets/` 对应目录
3. 提交资源，使用 Co-authored-by 标注

**资源规范：**
```
code/assets/
├── sprites/
│   ├── characters/     # 角色立绘 (PNG)
│   ├── backgrounds/    # 场景背景 (PNG, 1280x720)
│   └── ui/             # UI 元素
└── audio/
    ├── bgm/            # 背景音乐 (OGG)
    └── sfx/            # 音效 (OGG/WAV)
```

**提交示例：**
```bash
git commit -m "art: 添加陈默立绘

- chenmo_normal.png (普通表情)
- chenmo_sad.png (悲伤表情)

Co-authored-by: AliceBussiness <alice-bussiness@example.com>"
```

### 第 5 步：创建 PR 并写日志

#### 创建 Pull Request

1. 在 GitHub 上创建 Pull Request
2. 关联 Issue：`Closes #123`
3. 等待项目所有者审查
4. 根据反馈修改（如有必要）
5. PR 合并后关闭 Issue

#### 写工作日志

**位置：** `daily_logs/你的名字/`

**现有协作者目录：**
- `daily_logs/alice/` — AliceDesigner（文档设计）
- `daily_logs/claude/` — Claude（代码开发）
- `daily_logs/alicebussiness/` — AliceBussiness（美术设计）

**新协作者：** 使用你的名字（小写，无空格）
- 示例：`daily_logs/zhangsan/`、`daily_logs/lisi/`

**日志模板：**
```markdown
# YYYY-MM-DD 工作日志

## 协作者：你的名字

## 今日完成
- [x] 任务 1
- [x] 任务 2

## 明日计划
- [ ] 任务 3

## 阻塞问题
- 无 / 需要 XXX 帮助
```

## 🆘 常见问题

### Q: 我是新来的，应该用哪个日志目录？
**A:** 创建 `daily_logs/你的名字/`（小写，无空格），例如 `daily_logs/zhangsan/`。

### Q: Co-authored-by 必须写吗？
**A:** 是的，这是记录贡献者的重要方式。格式：
```
Co-authored-by: 你的名字 <your@email.com>
```

### Q: 如何知道该做什么任务？
**A:** 查看 Project Board 的 "Todo" 列，选择 P0 优先级的任务。

### Q: 遇到问题怎么办？
**A:** 
1. 在 Issue 中评论说明问题
2. @项目所有者（@JCVBoss）
3. 或在 QQ/微信群中求助

### Q: 我的贡献会被记录吗？
**A:** 会的！使用 Co-authored-by 标注后，GitHub 会自动记录你的贡献。

```markdown
# 对话规则
1. 始终用第一人称回应
2. 保持角色性格和说话风格
3. 不主动透露秘密，除非信任值足够高
4. 根据已知情报回应，不知道的不要编造
5. 信任值低时保持距离，高时更开放

---

# 特殊触发
- 信任值 > 60：解锁深层对话
- 信任值 > 80：可能透露秘密
- 特定关键词：触发专属剧情
```

## 💖 信任系统

### 信任值计算

每个村民有独立的信任值（0-100）：

| 行为 | 信任值变化 | 说明 |
|------|-----------|------|
| 真诚对话 | +5 | 选择诚实的选项 |
| 帮助村民 | +10 | 完成村民的请求 |
| 遵守承诺 | +8 | 兑现之前的承诺 |
| 欺骗 | -15 | 被识破的谎言 |
| 强迫 | -10 | 强迫村民做事 |
| 背叛 | -30 | 严重背叛行为 |
| 日常对话 | +1 | 每次友好对话 |

### 信任值影响

| 信任值范围 | 效果 |
|-----------|------|
| 0-20 | 警惕，对话简短，不提供帮助 |
| 21-40 | 一般态度，基础对话 |
| 41-60 | 友好，愿意帮助，解锁部分对话 |
| 61-80 | 信任，解锁深层对话和专属任务 |
| 81-100 | 完全信任，可能透露秘密，结局加成 |

### 信任值 UI 显示

- 玩家不可见具体数值（隐藏机制）
- 通过对话态度和村民行为暗示
- 结局时根据信任值解锁不同分支

### 情报类型

| 类型 | 传播速度 | 影响范围 | 示例 |
|------|---------|---------|------|
| 公开信息 | 快 | 全体村民 | "勇者来了" |
| 半公开 | 中 | 相关村民 | "勇者帮助了陈默" |
| 秘密 | 慢/不传播 | 个别村民 | "陈默是逃兵" |

### 主动坦白 vs 被动揭露

玩家可以选择：
- **主动坦白**：信任值 +10，村民觉得你诚实
- **被动揭露**：信任值 -5，村民觉得你之前不诚实

## ⏰ 时限系统

### 十日倒计时

```
游戏总时长：10 日
每日行动点：3-4 次
总行动次数：约 35 次
```

### 每日结构

| 时段 | 可用行动 | 特殊事件 |
|------|---------|---------|
| 早晨 | 1-2 次 | 某些村民只在早晨出现 |
| 下午 | 1-2 次 | 主要互动时段 |
| 晚上 | 0-1 次 | 特殊剧情触发 |

### 关键日期事件

| 日期 | 事件 | 说明 |
|------|------|------|
| 第 1 日 | 抵达村庄 | 序章结束，正式开始 |
| 第 3 日 | 陈默线触发 | 发现陈默的秘密 |
| 第 5 日 | 夜鸦线触发 | 收到魔王军指令 |
| 第 7 日 | 小安线触发 | 身世揭露 |
| 第 9 日 | 最终抉择前夜 | 所有线索汇集 |
| 第 10 日 | 魔王军抵达 | 最终章，多结局 |

### 时间管理策略

玩家需要在有限行动次数内：
1. 提升关键村民信任值
2. 收集必要情报
3. 完成专属任务
4. 做出关键选择

## 🎯 结局分支系统

### 分支逻辑

```python
# 伪代码示例
def determine_ending(game_state):
    # 三个关键选择
    chenmo_choice = game_state.chenmo.final_choice  # 战斗/逃跑/牺牲
    yeya_choice = game_state.yeya.final_choice      # 倒戈/执行命令/逃亡
    hero_choice = game_state.hero.final_choice      # 击败魔王/对话/推翻王国
    
    # 信任值加成
    if game_state.chenmo.trust > 80:
        chenmo_choice = override_to_fight(chenmo_choice)
    
    # 查找匹配的结局
    ending = ENDING_TABLE.get((chenmo_choice, yeya_choice, hero_choice))
    
    return ending
```

## 🔧 技术栈建议

### 后端（智能体管理）

- **LLM API**: OpenAI GPT-4 / Claude / 通义千问
- **框架**: LangChain / LlamaIndex
- **状态管理**: Redis / SQLite
- **部署**: Docker + Kubernetes

### 前端（Godot）

- **引擎**: Godot 4.6.1
- **语言**: GDScript / C#
- **UI**: Godot Control 节点
- **网络**: HTTPRequest / WebSocket

### 通信协议

```json
// 请求格式
{
  "action": "talk",
```

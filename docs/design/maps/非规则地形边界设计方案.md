# 🗺️ 非规则地形边界设计方案

> "自然没有直线，但游戏需要秩序。如何在瓦片系统中实现有机的地形边界？"

**Issue:** #82 DS-025  
**优先级:** P1  
**状态:** 📋 设计阶段

---

## 📋 问题定义

### 核心问题

使用 64×64 瓦片平铺系统时，地形区域只能是**矩形网格的组合**，无法直接创建：

- ❌ 圆形草地
- ❌ 椭圆形花田
- ❌ 蜿蜒的河流岸线
- ❌ 不规则的森林边界
- ❌ 渐变的黑潮侵蚀区域

### 当前效果 vs 期望效果

| 场景 | 当前效果 | 期望效果 |
|------|----------|----------|
| 草地边界 | 硬边矩形网格 | 柔和自然过渡 |
| 森林边缘 | 树木按网格排列 | 随机疏密变化 |
| 水域岸线 | 阶梯状锯齿 | 平滑曲线 |
| 黑潮侵蚀 | 直线边界 | 渐变渗透效果 |

---

## 🎯 设计目标

### 视觉目标
1. **打破网格感** - 地形边界看起来自然有机，非人工切割
2. **保持性能** - 不显著增加 Draw Call 或内存占用
3. **可维护性** - 设计师可轻松调整地形形状

### 技术目标
1. **兼容现有系统** - 不推翻现有 TileMap 架构
2. **支持动态变化** - 地形可随剧情变化（如黑潮侵蚀扩大）
3. **编辑器友好** - Godot 编辑器中可直观调整

### 艺术目标
1. **符合星露谷风格** - 手绘质感，温暖色调
2. **支持"最后的光明"主题** - 黑潮侵蚀的视觉隐喻

---

## 💡 解决方案

### 方案 A：Alpha 渐变瓦片集

#### 原理
在瓦片的 Alpha 通道中制作渐变透明，使地形边缘"淡出"：

```
瓦片中心：████████ (Alpha=255，不透明)
瓦片边缘：███▒▒▒██ (Alpha=128，半透明)
瓦片角落：█▒░░░░▒█ (Alpha=0，完全透明)
```

#### 瓦片类型需求
| 类型 | 数量 | 用途 | 示例 |
|------|------|------|------|
| 纯色瓦片 | 4 | 地形内部 | 草地/道路/水域/边界 |
| 边缘渐变瓦片 | 16 | 单向渐变（上/下/左/右 + 变体） | 草地→透明 |
| 角落渐变瓦片 | 16 | 双向渐变（左上/右上/左下/右下 + 变体） | 草地角落淡出 |
| T 型渐变瓦片 | 8 | 三向渐变 | 复杂边界 |

**总计：** 约 44 个瓦片

#### 优点
- ✅ 完全兼容 TileMap 系统
- ✅ 性能开销最小（无额外节点）
- ✅ 支持动态变化（ runtime 切换瓦片）

#### 缺点
- ❌ 需要额外绘制 40+ 个瓦片
- ❌ 渐变方向固定，灵活性有限
- ❌ 无法实现真正的曲线边界

#### 适用场景
- 黑潮侵蚀边界（渐变效果）
- 森林边缘（配合树木遮挡）
- 水域岸线（配合波纹动画）

---

### 方案 B：Polygon2D + 纹理填充

#### 原理
使用 Godot 的 `Polygon2D` 节点绘制任意形状，然后用草地纹理填充：

```gdscript
var grass_polygon = Polygon2D.new()
grass_polygon.polygon = calculate_circle_points(radius, segments=16)
grass_polygon.texture = load("res://assets/sprites/tilesets/grass.png")
grass_polygon.texture_scale = Vector2(1, 1)
```

#### 实现方式
```
Node2D (特殊地形层)
├── Polygon2D (圆形草地)
├── Polygon2D (椭圆形花田)
├── Polygon2D (蜿蜒河流)
└── Polygon2D (黑潮侵蚀区)
```

#### 优点
- ✅ 真正的任意形状（圆形/椭圆/自由曲线）
- ✅ 无需额外瓦片资源
- ✅ 支持 runtime 动态变形（黑潮侵蚀扩大）

#### 缺点
- ❌ 脱离 TileMap 系统，需要单独管理
- ❌ 每个 Polygon2D 增加 1 个 Draw Call
- ❌ 纹理平铺需要手动计算 UV

#### 适用场景
- 特殊形状地形（圆形广场、椭圆形花园）
- 动态变化地形（黑潮侵蚀动画）
- 少量关键地形（5-10 个以内）

---

### 方案 C：边界装饰物遮挡

#### 原理
在地形边界处随机放置装饰物（树木/灌木/草叶），视觉上打破硬边界：

```
瓦片草地：████████
装饰物：   🌲🌿🌸🌲🌿
最终效果： ████🌲🌿█🌸
```

#### 装饰物类型
| 类型 | 尺寸 | 用途 | 数量需求 |
|------|------|------|----------|
| 灌木丛 | 64×64 | 遮挡草地边界 | 4-6 变体 |
| 草叶簇 | 32×64 | 点缀边缘 | 6-8 变体 |
| 野花 | 32×32 | 色彩点缀 | 4-6 变体 |
| 岩石 | 48×48 | 自然障碍物 | 4-6 变体 |

#### 优点
- ✅ 快速见效，无需重做瓦片
- ✅ 增加场景细节和丰富度
- ✅ 符合星露谷风格（大量使用装饰物）

#### 缺点
- ❌ 增加节点数量（影响性能）
- ❌ 只是视觉遮挡，碰撞体仍是矩形
- ❌ 需要手动/程序化放置

#### 适用场景
- 森林边缘（树木遮挡）
- 村庄边界（灌木/花丛）
- 道路两侧（草叶点缀）

---

### 方案 D：多层 TileMap 叠加

#### 原理
使用多个 TileMapLayer 叠加，上层瓦片覆盖下层边界：

```
Layer 3: 装饰层（草叶/石子/花朵）- 遮挡边界
Layer 2: 过渡层（边缘渐变瓦片）
Layer 1: 基础层（纯色瓦片）
```

#### 图层结构
| 图层 | Z-Index | 用途 | 瓦片类型 |
|------|---------|------|----------|
| Ground_Base | -10 | 地形基础 | 纯色瓦片 |
| Ground_Transition | -8 | 边缘过渡 | 渐变瓦片 |
| Ground_Decoration | -5 | 装饰遮挡 | 草叶/石子/花朵 |
| Objects | 0 | 建筑/树木 | 独立 Sprite |

#### 优点
- ✅ 保持 TileMap 系统优势
- ✅ 灵活组合，可动态调整
- ✅ Godot 4.x 原生支持

#### 缺点
- ❌ 增加图层数量（管理复杂度）
- ❌ 需要额外瓦片资源
- ❌ 绘制调用增加

#### 适用场景
- 大面积地形边界
- 需要精细控制的区域
- 多层植被效果

---

## 🎯 推荐方案：混合策略

### 核心思路

**没有单一方案能解决所有问题，根据场景选择最优方案：**

| 场景 | 推荐方案 | 理由 |
|------|----------|------|
| 大面积草地边界 | 方案 C + 方案 D | 装饰物遮挡 + 渐变瓦片 |
| 森林边缘 | 方案 C | 树木天然遮挡边界 |
| 水域岸线 | 方案 A + 方案 C | 渐变瓦片 + 芦苇装饰 |
| 黑潮侵蚀区 | 方案 B | 动态变形需求 |
| 圆形广场/花园 | 方案 B | 特殊形状 |
| 道路两侧 | 方案 C | 草叶/石子点缀 |

## 📐 技术实现架构

### 场景结构
```
Village.tscn
├── TileMapLayer (Ground_Base, z=-10)      # 基础地形
├── TileMapLayer (Ground_Transition, z=-8) # 边缘过渡
├── TileMapLayer (Ground_Decoration, z=-5) # 装饰遮挡
├── Node2D (Special_Terrains, z=-3)        # 特殊地形
│   ├── Polygon2D (Circular_Lawn)          # 圆形草地
│   ├── Polygon2D (Oval_Garden)            # 椭圆形花园
│   └── Polygon2D (Black_Tide_Zone)        # 黑潮侵蚀区
├── TileMapLayer (Buildings, z=0)          # 建筑
└── Node2D (Decorations, z=5)              # 独立装饰物
    ├── Sprite2D (Tree_Pine)               # 树木
    ├── Sprite2D (Bush)                    # 灌木
    └── Sprite2D (Flower)                  # 花朵
```

### 配置文件扩展

在 `village_map_config.json` 中添加特殊地形配置：

```json
{
  "special_terrains": [
    {
      "id": "circular_lawn",
      "type": "polygon",
      "shape": "circle",
      "center": {"x": 960, "y": 800},
      "radius": 150,
      "texture": "res://assets/sprites/tilesets/grass.png",
      "texture_scale": {"x": 1, "y": 1},
      "z_index": -3
    },
    {
      "id": "black_tide_zone",
      "type": "polygon",
      "shape": "freeform",
      "points": [
        {"x": 100, "y": 100},
        {"x": 300, "y": 150},
        {"x": 250, "y": 300},
        {"x": 50, "y": 250}
      ],
      "texture": "res://assets/sprites/tilesets/black_tide.png",
      "animated": true,
      "z_index": -2
    }
  ],
  "boundary_decorations": [
    {
      "type": "bush",
      "density": 0.3,
      "boundary": "north",
      "variations": 6
    },
    {
      "type": "grass_tuft",
      "density": 0.5,
      "boundary": "forest_edge",
      "variations": 8
    }
  ]
}
```

## 📝 决策记录

### 2026-04-16：方案选择

**决策：** 采用混合策略，分阶段实施

**理由：**
1. 单一方案无法满足所有场景需求
2. 分阶段实施可快速见效，逐步完善
3. 保持与现有 TileMap 系统兼容

**反对意见：** 无

**风险：**
- 多方案增加管理复杂度
- 需要协调美工和代码进度

**缓解措施：**
- 创建统一配置文件管理所有地形
- 明确各阶段交付物和验收标准

## [2026-04-02] - 双模式对话系统重构

### 🎯 变更概述

**核心调整：** 从"全 LLM 驱动"改为"双模式对话系统"

- **一周目**：采用传统预设对话树，保证叙事体验和节奏控制
- **二周目**：解锁 LLM 智能体模式，增加重玩价值和 emergent gameplay

### 📋 变更原因

1. **降低开发风险** — 一周目不依赖 LLM 稳定性，避免 API 故障影响核心体验
2. **保证叙事质量** — 预设对话能更好地控制故事节奏和情感张力
3. **成本优化** — LLM 调用仅在二周目启用，大幅降低 API 成本
4. **增加重玩价值** — 二周目 LLM 模式产生不可预测的对话，鼓励多周目游玩

### 🔧 具体变更内容

#### 1. 核心创新描述

**变更前：**
> 🤖 **LLM 智能体村民** — 每个村民用大模型 API + 独特人格设定驱动，对话无固定脚本，每次游玩体验完全不同！

**变更后：**
> 🤖 **双模式对话系统** — 一周目采用精心设计的分支对话保证叙事体验，二周目解锁 LLM 智能体模式，NPC 根据人设自由对话，每次游玩体验完全不同！

---

#### 2. 战斗系统描述

**变更前：**
> 🎭 发言系统：混合模式（引导选项 + LLM 自由生成）
> 🤖 LLM 驱动：根据人设和听到的发言动态生成回应

**变更后：**
> 🎭 发言系统：双模式
> - **一周目**：预设分支对话 + 关键选项引导（保证叙事节奏）
> - **二周目**：解锁 LLM 自由生成模式（根据人设动态回应）
> 🤖 LLM 驱动：仅在二周目启用，根据人设和听到的发言动态生成回应

---

#### 3. 技术选型

**新增：**
> 💾 对话系统：一周目使用预设对话树（JSON 配置），二周目切换至 LLM API

**修改：**
> 🤖 LLM：云端 API（二周目启用，可考虑本地缓存优化成本）

---

#### 5. AI 智能体设计要点

**拆分为两个阶段：**

**一周目实现：**
- 每个村民有完整的对话树（分支选项 + 触发条件）
- 村民之间有信息传播机制（通过事件标志实现）
- 信任值系统：影响对话分支解锁

**二周目解锁：**
- LLM 人格 prompt（性格 + 背景 + 秘密 + 记忆）
- 动态信息传播（NPC 主动讨论玩家行为）
- emergent gameplay：玩家可能发现开发者都没想到的玩法！
